# OrchestRouter UI - 快速使用指南

## 新增功能：灵活的模型提供商配置

### 1. 启动服务器
```bash
# 启动后端服务器
cd src/orchestrator
node OrchestratorServer.js

# 启动前端服务器
cd ui
npm run dev
```

### 2. 使用自定义模型提供商

#### 方法一：使用预设模板
1. 访问 UI 界面（通常在 http://localhost:5173）
2. 点击"模型提供商"标签页
3. 从下拉菜单中选择提供商类型（如 OpenAI、Anthropic 等）
4. 点击"添加模板"按钮
5. 配置提供商信息

#### 方法二：添加完全自定义提供商（新增功能）
1. 访问 UI 界面
2. 点击"模型提供商"标签页
3. 点击"添加自定义"按钮
4. 填写自定义提供商的信息：
   - 提供商名称
   - API Base URL
   - API 环境变量名
   - 硬编码 API 密钥（可选）
5. （可选）在"高级配置"部分配置：
   - 供应商转换器
   - 自定义请求头

### 3. 高级配置选项

#### 供应商转换器
用于指定特定于供应商的请求/响应转换器，帮助适配不同供应商的API格式。

#### 自定义请求头
以JSON格式提供额外的请求头，例如：
```json
{
  "X-Custom-Header": "value",
  "Authorization": "Bearer your-token"
}
```

### 4. 保存配置
配置完成后，点击右上角的"保存配置"按钮。

现在您可以更灵活地配置各种模型提供商，不再局限于预设的模板！
